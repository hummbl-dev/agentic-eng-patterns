# v0.1 Scope and Boundary

This document defines the scope, non-goals, adoption boundary, review expectations,
non-canon posture, and required packet pieces for v0.1 of the agentic engineering
patterns seed repository. It closes issue #1.

## Status

Candidate. Nothing in v0.1 is adopted guidance. v0.1 is an exploratory baseline
intended to make the domain reviewable and machine-checkable without claiming
authority over the broader field.

## Scope

v0.1 covers the minimum artifacts needed to describe agentic engineering patterns
as version-controlled, reviewable, testable, auditable, and agent-operable source
material.

In scope for v0.1:

- A written definition of the domain (`docs/definition.md`).
- A set of operating principles (`docs/principles.md`).
- This scope and boundary document (`docs/v0.1-boundary.md`).
- A curated set of public prior art and adjacent ecosystem references
  (`docs/prior-art.md`).
- A minimal JSON Schema for an agentic patterns packet
  (`schemas/agentic-eng-patterns-v0.1.json`).
- A valid example packet (`examples/agentic-pattern-packet-v0.1.example.json`).
- Valid and invalid fixtures used to exercise the schema
  (`fixtures/valid/`, `fixtures/invalid/`).
- A receipt that records what was produced and why
  (`receipts/agentic-eng-patterns-v0.1-packet-receipt.md`).

Out of scope for v0.1:

- A universal standard or specification.
- Adopted, governance-approved guidance.
- A runtime, SDK, or execution environment.
- Private, internal, or secret operational content.
- Canonization of any vendor, product, or internal concept.

## Non-goals

- This repository is not a universal standard.
- This repository does not claim ownership of the broader field or terminology.
- This repository does not canonize HUMMBL/BaseN/Ownward or any other vendor
  concepts unless explicitly marked and audited.
- This repository must not include private, internal, or secret operational
  content.
- v0.1 does not define adoption criteria, conformance tests, or a maturity model.
  Those are deferred to later versions per `docs/roadmap.md`.

## Adoption boundary

v0.1 is **candidate namespace only**. Adoption requires a reviewed governance
path that does not yet exist in this repository. Until that path is defined:

- No artifact in v0.1 should be treated as normative.
- No artifact in v0.1 should be cited as a standard.
- Consumers may use the schema and examples as a starting point, but any
  production use is at the consumer's own risk and must be reviewed locally.
- Forks and derivative work are welcome; this repository makes no claim of
  control over derivative terminology.

## Review expectations

v0.1 artifacts are intended to be small, reviewable, and evidence-backed. A
reviewer of v0.1 should be able to check:

- That scope and non-goals are stated explicitly.
- That prior art references point to public, linkable documentation.
- That the schema is minimal, internally consistent, and round-trips against the
  provided valid and invalid fixtures.
- That the example packet satisfies the schema.
- That the invalid fixture fails the schema for a stated, narrow reason.
- That no private, internal, or secret operational content is present.
- That no vendor concept is canonized without an explicit, audited marker.

## Non-canon posture

This repository explicitly avoids canonization. Concretely:

- Naming a tool, framework, or concept in `docs/prior-art.md` is a reference,
  not an endorsement or adoption.
- Vocabulary terms (agent, tool, planning, reflection, memory, delegation,
  orchestration, multi-agent, handoff) are used descriptively and are not
  claimed as proprietary definitions.
- Any future canonization of a pattern, term, or vendor concept must go through
  a reviewed governance path and must carry an explicit, audited marker in the
  artifact and in the receipt.

## Required packet pieces

A v0.1 agentic patterns packet must contain, at minimum, the following pieces
in order to be considered complete for review:

1. **Schema** — `schemas/agentic-eng-patterns-v0.1.json` defines the structural
   contract for a packet.
2. **Example** — `examples/agentic-pattern-packet-v0.1.example.json` is a
   valid, human-readable instance of the schema.
3. **Valid fixture** — `fixtures/valid/agentic-pattern-packet-v0.1.valid.json`
   is a minimal instance that must pass schema validation.
4. **Invalid fixture** — `fixtures/invalid/agentic-pattern-packet-v0.1.invalid.json`
   is an instance that must fail schema validation for a stated, narrow reason
   (a missing required field).
5. **Receipt** — `receipts/agentic-eng-patterns-v0.1-packet-receipt.md`
   records what was produced, which issues it closes, and the non-canon posture.

A packet that omits any of these pieces is incomplete for v0.1 review purposes.
